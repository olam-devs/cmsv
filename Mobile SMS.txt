Mobile SMS
Send message using Android-based SMS gateway directly from multiple SIM cards and Android phones. Message is sent using SIM card number.

Registered gateway accounts are needed before sending message. To register gateway account, login to your dashboard, Channels (Menu) -> Mobile Channel (Menu) -> Android Gateway (Menu)

If you have multiple active gateway accounts and sim card numbers, system will distribute messages to all sim cards in round-robin algorithm.

All users subscribed to the mobile network which your SIM card number can send message will receive message.

AUTHORIZATION
Bearer Token
This folder is using Bearer Token from collectionMessaging Service API V2
Send Sms
In a few simple steps, we will explain how to send an SMS using HTTPS API.

Firstly, you'll need a valid account. When you sign up for the account, you will set a username and password. Next, while logged to your account navigate to Customer Profile (Top Menu Right) -> Customization (Tab) -> API Keys (Tab) to get access token as described in the Authorization Section:

The message will be sent only to a valid phone number (numbers), written in international format e.g. 255738234345. We strongly recommend using the E.164 number formatting. E.164 numbers are internationally standardized to a fifteen digit maximum length. Phone numbers are usually prefixed with + (plus sign), followed by a country code, network code and the subscriber number. Phone numbers that are not E.164 formatted may work, depending on the handset or network.

When sending a message you can specify reference value which associates the API request payload with your data. This is very useful when fetching delivery reports and logs. This is also useful when at any circumstance the requester does not receive a response from the API endpoint and wants to synchronise information at some point later.

GET
Get all sent SMS logs with optional parameter
https://messaging-service.co.tz/api/v2/logs?from=TANZANIATIP&to=255716718040&sentSince=2025-03-01&sentUntil=2025-05-21
you might need some filters to search through them. The filters you can use are:

from:
Sender ID name.

to:
Destination (Phone number starting with 255).

sentSince: Date.
Lower limit on date and time of sending SMS.

sentUntil: Date.
Upper limit on date and time of sending SMS.

offset: To skip a given number of results in the report. This must be an integer.

limit: To limit the number of results returned use limit, this must be an integer (Maximum allowed is 500).

Using parameters is optional you can just use the provided API url to get all of your sent sms logs.

reference:

This is the special value which was used during sending a message to associate the sent message with specific API request values.
